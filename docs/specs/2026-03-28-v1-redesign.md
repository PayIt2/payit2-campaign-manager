# PayIt2 Campaign Assistant v1.0.0 — Redesign Spec

**Date:** 2026-03-28
**Author:** Brian Anderson + Claude
**Status:** Draft

---

## Overview

Redesign the Campaign Assistant plugin from a fundraising-first tool with bolted-on event/collection support into a unified campaign coaching platform. All commands, skills, and agents work across all 3 campaign types: fundraisers, events, and group collections.

**Target platform:** Claude Desktop App (conversational-first)
**Version:** 1.0.0
**No MCP this release** — all context gathered through user conversation. Architecture should be MCP-ready for a future release.

---

## Architecture

Three layers: commands (user-facing), skills (capability logic), agents (heavy-lift autonomous work).

```
COMMANDS (4)
  /campaign    — Create & launch any campaign type
  /promote     — Drive traffic, shares, and ticket sales
  /check-in    — Weekly health check with diagnostics
  /engage      — Supporter relationships and communications

SKILLS (5)
  campaign-context         — Shared context-gathering engine
  campaign-creation        — Page building workflow
  campaign-promotion       — Multi-channel promotion strategy
  campaign-analytics       — Health scoring and diagnostics
  supporter-engagement     — Thank-yous, updates, re-engagement

AGENTS (3)
  campaign-assistant           — Strategic analysis (Opus)
  content-generator        — Batch content generation (Sonnet)
  supporter-outreach       — Relationship communications (Sonnet)
```

**Design principle:** Every command, skill, and agent works for all 3 campaign types. Type-specific advice (ticket tiers for events, reminder cadence for collections, goal psychology for fundraisers) lives as branches within each component, not as separate components.

## Commands

### `/campaign` — Create & Launch

The front door. Walks the user through building a campaign page from scratch.

**Flow:**
1. **Context gather:** "What are you creating?" → fundraiser, event, or group collection
2. **Story/details interview** — adapts questions by type:
   - **Fundraiser:** Who needs help, what happened, what funds cover, timeline, stakes
   - **Event:** Type, date/time, location, capacity, what attendees experience
   - **Group collection:** Purpose, group size, total amount, deadline, split model
3. **Title generation** — 5 options scored on clarity, emotion, SEO (under 60 chars)
4. **Description writing** — adapted structure by type:
   - Fundraiser: Hook → Person → Situation → Plan → Ask (~150 words)
   - Event: Hook → What/When/Where → Who It's For → What's Included → Schedule → Logistics (<200 words)
   - Collection: Purpose → Amount breakdown → Deadline → How to pay
5. **Visual strategy** — what photos/video to use
6. **Goal/pricing setup:**
   - Fundraiser: goal psychology (under $5K = 2.5x more likely to succeed)
   - Event: ticket tier design (2-4 tiers, 20-25% price gaps, early bird strategy)
   - Collection: cost breakdown (fixed split, tiered options, or flexible/voluntary)
7. **Pre-publish checklist** — type-specific verification
8. **Output:** Launch-ready package saved to markdown

**Triggers:** "create a campaign", "start a fundraiser", "set up an event", "collect from my group", "create a page", "launch a campaign", "new campaign", "set up my PayIt2 page"

### `/promote` — Drive Traffic & Shares

Multi-channel promotion strategy and content generation for any campaign type.

**Flow:**
1. **Context gather:** "Tell me about your campaign" — type, title, URL, audience, how long it's been live, what you've tried
2. **Assess current state:** What channels have you used? What's working?
3. **Channel strategy** — prioritized by type and network:
   - Fundraisers: lean on personal networks (Facebook, email, text)
   - Events: lean on social discovery (Instagram, TikTok, community boards)
   - Collections: lean on direct messaging (group text, Slack/Teams, email)
4. **Content calendar** — phased by lifecycle:
   - Fundraiser: Launch Blitz → Momentum → Sustain → Final Push (30 days)
   - Event: Announce → Early Bird → Social Proof → Last Chance → Day-Of → Post-Event (4-6 weeks)
   - Collection: Launch → Progress → Midpoint → Urgency → Deadline (2-3 weeks)
5. **Generate batch content** — platform-native posts, emails, text messages
6. **Output:** Content package saved to markdown

**Triggers:** "promote my campaign", "create social media posts", "build a content calendar", "write emails for my campaign", "how do I get more donations", "sell more tickets", "increase visibility", "campaign marketing", "share my fundraiser", "promote my event"

### `/check-in` — Weekly Health Check

Diagnoses performance and prescribes actions for any campaign type.

**Flow:**
1. **Context gather:** "How's your campaign going? Share your current numbers."
2. **Prompt for key stats** by type:
   - Fundraiser: Amount raised, donor count, days active, recent sharing activity
   - Event: Tickets sold, capacity %, days until event, registration velocity
   - Collection: Payments received, people paid vs total, days until deadline
3. **Calculate health score** (0-100) with type-adapted weights:
   - Fundraiser weights: Momentum 25%, Social reach 20%, Story quality 15%, Donor engagement 15%, Goal progress 15%, Network activation 10%
   - Event weights: Registration velocity 25%, Capacity utilization 20%, Promotion reach 20%, Engagement 15%, Revenue 10%, Logistics readiness 10%
   - Collection weights: Payment rate 30%, Deadline proximity 20%, Communication cadence 20%, Response rate 15%, Transparency 15%
4. **Diagnose bottleneck:** Traffic problem, conversion problem, or engagement problem
5. **This week's action plan** — 3-5 prioritized actions
6. **Generate content** if prescribed
7. **Output:** Weekly report saved to markdown

**Score interpretation (all types):**
- 80-100: Thriving — maintain cadence, expand
- 60-79: Healthy — address 1-2 weak areas
- 40-59: At risk — significant changes needed (dispatches campaign-assistant agent)
- 20-39: Critical — major intervention needed (dispatches campaign-assistant agent)
- 0-19: Stalled — consider relaunch or pivot

**Triggers:** "check my campaign health", "how is my campaign doing", "why aren't I getting donations", "why aren't tickets selling", "campaign is stalled", "weekly check-in", "what should I do next", "campaign analytics", "campaign optimization"

### `/engage` — Supporter Relationships

Thank-yous, updates, re-engagement, and advocacy for any campaign type.

**Flow:**
1. **Context gather:** "What would you like to do?" — thank supporters, post an update, re-engage lapsed supporters, ask for shares
2. **Adapt by type and action:**
   - **Thank supporters:**
     - Fundraiser: Donor thank-yous by tier (micro $1-25, standard $26-100, major $101-500, anchor $500+)
     - Event: Attendee confirmation + excitement builder
     - Collection: Payment acknowledgment + progress update
   - **Post update:**
     - Fundraiser: Milestone updates (25%, 50%, 75%, goal), impact reports
     - Event: Lineup reveals, behind-the-scenes, logistics updates
     - Collection: Progress tally ("12 of 20 paid"), deadline reminder
   - **Re-engage:**
     - Fundraiser: Story updates for lapsed donors, share requests
     - Event: Last-chance messaging for non-registrants
     - Collection: Private 1:1 follow-up for non-payers (Friendly Collector principles)
   - **Ask for shares:**
     - All types: Pre-written share content, co-organizer invitations, donor/attendee challenges
3. **Generate personalized messages** across channels (email, text, social, in-campaign)
4. **Output:** Engagement package saved to markdown

**Triggers:** "thank my donors", "thank my supporters", "write thank-you messages", "post a campaign update", "show impact", "re-engage supporters", "follow up with attendees", "remind people to pay", "donor retention", "attendee follow-up"

## Skills

### `campaign-context`

Shared context-gathering engine used by every command. Responsible for figuring out what the user is working on without an MCP.

**Behavior:**
- Starts every command interaction with a quick context check
- Asks the minimum questions needed for the current command
- Adapts questions based on what's already known in the conversation
- Can accept a PayIt2 page URL as a shortcut — extracts what it can from URL structure, asks for the rest
- No persistent file storage — context lives in the conversation (Claude Desktop)

**Context model:**

| Field | Needed By | Required/Optional |
|-------|-----------|-------------------|
| Campaign type (fundraiser/event/collection) | All | Required |
| Title | All | Required |
| URL (if live) | promote, check-in, engage | Optional |
| Goal or target amount | campaign, check-in | Required for fundraiser/collection |
| Ticket tiers / pricing | campaign, check-in | Required for events |
| Current progress (amount/tickets/payments) | check-in, engage | Required for check-in |
| Days active / days until deadline or event | check-in, promote | Required for check-in |
| Audience description | promote, campaign | Optional |
| Channels used so far | promote, check-in | Optional |
| Group size | campaign, check-in | Required for collections |

**MCP-readiness:** When an MCP is added in a future release, this skill becomes the adapter layer. Instead of prompting the user, it queries the MCP for campaign data. The rest of the skills don't change — they receive context the same way regardless of source.

### `campaign-creation`

Guides building the campaign page. Contains all type-specific creation knowledge.

**Contents:**
- Story interview frameworks by type (fundraiser story arc, event details, collection breakdown)
- Title formulas and scoring rubric (clarity, emotional impact, SEO value)
- Description templates adapted by type
- Visual strategy guidance (hero image, supporting photos, video recommendations)
- Goal/pricing psychology:
  - Fundraiser: Goals under $5K are 2.5x more likely to succeed, 20-30% in week 1 = 80% higher success
  - Event: Early bird 25-30% off for first 30% capacity, 2-4 tiers with 20-25% gaps, group discounts
  - Collection: Fixed split, tiered options, or flexible/voluntary models
- Pre-publish checklists by type
- Research statistics (co-organizer 3x, video 4x, title "help" 30%, description ~150 words optimal)

**Reference docs:**
- `story-templates.md` — category-specific frameworks (legal defense, medical, emergency, education, events, collections)
- `title-formulas.md` — proven patterns with scoring, expanded for events and collections
- `ticket-strategy.md` — tier templates, pricing formulas, group discounts
- `cost-splitting-guide.md` — fixed/tiered/flexible models, fee transparency, overflow handling

### `campaign-promotion`

Unified promotion engine across all campaign types.

**Contents:**
- The 2.5x Rule: each new sharing method = 2.5x visibility; 6+ methods = 3x more donations
- Channel priority matrix adapted by type
- Content calendar frameworks:
  - Fundraiser: 4 phases over 30 days (Launch Blitz, Momentum, Sustain, Final Push)
  - Event: 6 phases over 4-6 weeks (Announce, Early Bird, Social Proof, Last Chance, Day-Of, Post-Event)
  - Collection: 5 phases over 2-3 weeks (Launch, Progress, Midpoint, Urgency, Deadline)
- Email outreach tiers (Tier 1 family → Tier 4 cold outreach)
- 8 content rotation angles with type equivalents
- Platform-specific formatting and best practices
- SEO optimization (keywords, meta descriptions, community posting angles — absorbed from seo-optimizer)
- Event-specific: 61% millennials discover via social, 4-6 week lead time = 70% more sales
- Collection-specific: max 5-6 messages to full group, private follow-ups only

**Reference docs:**
- `post-templates.md` — unified templates across all types and platforms
- `email-sequences.md` — drip sequences for all types (launch, updates, final push, event countdown, collection reminders)

### `campaign-analytics`

Health scoring and diagnostics adapted by campaign type.

**Contents:**
- Health score formula (0-100) with type-adaptive weight sets (see `/check-in` command)
- Score interpretation thresholds (thriving → stalled)
- 3-stage diagnostic framework (traffic → conversion → engagement)
- Type-specific KPIs:
  - Fundraiser: donation velocity >2/day week 1, share rate >5%, conversion 5-15%, avg donation $35-75
  - Event: registration velocity 10-15% capacity/week, early bird 25-35% of total, email open 40-60%
  - Collection: payment rate benchmarks by group size, response rate to reminders
- Category benchmarks (fundraiser categories from GoFundMe data + event type benchmarks + collection type benchmarks)
- Optimization playbook by effort level (quick wins, medium effort, strategic moves)

**Reference docs:**
- `benchmark-data.md` — detailed benchmarks by category and type
- `optimization-checklist.md` — comprehensive checklist unified across types

### `supporter-engagement`

The relationship layer across all campaign types.

**Contents:**
- Thank-you system with personalization tiers (works for donors, attendees, and group members)
- Thank-you framework: Name → Acknowledge → Show impact → Humanize → Extend
- Timing: <1hr personalized, <24hr public acknowledgment, <1 week impact update
- Update cadence by milestone type (25%, 50%, 75%, goal, post-campaign)
- Update content framework: Progress metric → Emotional beat → Fund usage → Forward look → CTA
- Impact reporting (before/after, receipt transparency, beneficiary voice)
- Re-engagement triggers and message types
- "Friendly Collector" principles (applied broadly, not just collections):
  - Frame reminders as progress updates, never accusations
  - Never single out non-participants publicly
  - Lead with gratitude and progress
  - Celebrate milestones publicly
- Donor/attendee/member-to-advocate conversion tactics

**Reference docs:**
- `thank-you-templates.md` — templates by tier and channel, expanded for all types
- `update-templates.md` — templates for each milestone type across all types
- `reminder-templates.md` — collection reminders and re-engagement messages (consolidated from collection-templates.md)

## Agents

### `campaign-assistant` (Opus)

Strategic analysis agent. Dispatched for deep diagnosis when a campaign is struggling.

**Dispatch triggers:** Health score below 60, user asks "why isn't this working", user requests honest assessment.

**Output structure:**
1. Campaign Snapshot (type, progress, days active, key metrics)
2. Health Score (0-100) with factor breakdown
3. Primary Diagnosis (the one thing that matters most)
4. What's Working (reinforce good habits)
5. Top 5 Actions (ranked by impact, specific and actionable)
6. 7-Day Game Plan (day-by-day actions)
7. Honest Forecast ("at current pace, you'll reach X by Y")

**Works for all types.** Adapts diagnosis framework by type.

### `content-generator` (Sonnet)

Batch content engine. Dispatched when a command needs to produce multiple pieces of content at once.

**Dispatch triggers:** `/promote` content generation, `/check-in` prescribes fresh content, user asks for a batch of posts.

**Output structure:**
- Content organized by platform (Facebook, Instagram, TikTok, Twitter/X, LinkedIn, Email, Text/WhatsApp)
- Each piece is platform-native (correct length, tone, format)
- Organized by timeline phase
- Includes SEO recommendations (keywords, community posting targets, media outreach angles)

**Works for all types.** Adjusts content angles — urgency for fundraisers, countdown/FOMO for events, progress/social-proof for collections.

### `supporter-outreach` (Sonnet)

Relationship communications agent. Dispatched when `/engage` has a batch of people to communicate with.

**Dispatch triggers:** `/engage` with multiple supporters, batch thank-you generation, complex communication sequence needed.

**Output structure:**
- Messages organized by recipient (or recipient tier)
- Multi-channel versions (email, text, DM, social post)
- Personalized per tier and relationship type
- Includes follow-up sequences where appropriate (event attendee journey, collection reminder cadence)

**Works for all types.** Adapts tone and content — gratitude for donors, excitement for attendees, friendly accountability for group members.

## Out of Scope

- **MCP server** — future release. Architecture is MCP-ready (campaign-context skill is the adapter layer).
- **Persistent context storage** — no file-based campaign profiles. Context lives in the conversation.
- **PayIt2 app integration** — no API calls to PayIt2. All data comes from user conversation.
- **New campaign types** — fundraisers, events, and group collections only. Adding types later should be straightforward given the unified architecture.
